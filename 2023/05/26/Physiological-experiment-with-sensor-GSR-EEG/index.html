The design below is intended for the investigation of how physiology correlates with music genres. It can also be applied in many other studies.

The overall design is as follows:

Equipment

Computer system

Linux Lite

Galvanic skin response sensor

Groove-GSR sensor gives the human resistance change. By detecting how skin conductivity changes, it can spot the emotion change. I connected it with an Arduino mini board(A2->SIG, GND->GND, 5v->VCC).
Open Arduino IDE, select the right board, and upload the following code:

Congrats! You now can monitor the data from the serial port and save the data flow in python:

def read_GSR(ser, name):
    try:
        global stop
        while not stop:
            #while True:
            if ser.in_waiting > 0:
                with open(name + '.txt', 'a') as file:
                    data = ser.readline().decode().strip()
                    file.write(data + '\n')
        print("GSR file is saved")
    except Exception as e:
        # Handle the exception here or print a custom error message
        print("An error occurred while reading GSR:", str(e))

EEG sensor

TGAM module
Because it is connected to a Bluetooth module, I had a hard time installing the Bluetooth driver on Linux. However, it can be connected by simply using the password of “0000”. And then if one wants to monitor the wave using the provided serial debugging assistant, one needs to transfer the data from vitural serial port (RFCOMM 0) to a normal serial port(COM 33):

sudo ln -s /dev/rfcomm0 /dev/ttyS32

If the monitor is not only of interest, one should process the data by oneself. In this case, it is not necessary to transfer:

Interface

Participants were asked to put on a headphone which connected to the computer. After putting the name and emotions in the terminal, music will be played and the data from sensors will be stored automatically. Except for the pulse oximeter, data from the other two can be fetched through a computer. As a result, three threads are running during the experiment to play the music , fetch data from GSR sensor and EEG sensor.

Plot the GSR sensor value while marking the start point of each song.

def plotGSR(fileID):
    with open(fileID+'_GSR.txt', 'r') as file:
        # Read the contents of the file
        contents = file.read()
        
        num_tem = re.findall(r'\d+', contents)
        
        # Convert the extracted numbers to integers
        num_list = [int(num) for num in num_tem]
        
        end_index= num_list[12:].index(2023)+12 #the last dataindex 
        
        X= [num for num in range(0, len(num_list[12:end_index]))]
        
        plt.plot(X,num_list[12:end_index],color="black", linewidth=2.0)
        
        
        ddot=len(X)/981
        ticks=[]
        label = list(range(1, 13))
        for i in range(10,900,80):#12 songs begin time
            ticks.append(i*ddot)
            
        plt.xticks(ticks,label)
        plt.grid(True)
        
        plt.savefig(fileID+'_GSR.svg',dpi=300,format="svg");
        
        plt.show();

from dealEEG import get_EEG,plot_EEG,plot_EEG_heatmap,plot_EEG_aver

name=['ca12','CH12','AG23',''];



for i in range(len(name)):
    data=get_EEG(name[i]);
    
    plot_1=plot_EEG(data,name[i]);
    
    plot_2=plot_EEG_heatmap(data,name[i]);
    
    plot_3=plot_EEG_aver(data,name[i]);

where get_EEG(name) returns the EEG data in a dictionary structure. Variables available are meditation level, attention level, brain wave and noise level. plot_EEG(data,name) plots attention, meditation and signal quality,plot_EEG_heatmap(data,name) plot the heat map of brain waves. plot_EEG_aver(data, name) gives the average value of the signal during one song.

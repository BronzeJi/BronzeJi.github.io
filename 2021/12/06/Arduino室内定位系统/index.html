Arduino室内定位器

一个通过识别WiFI信号名称与大小的室内房间定位器 。
对原系统进行了一些改动。

简介

  • 收集所在每个房间的数据，通过决策树算法学习，得到分类器
  • 再次实时输入数据时，将数据先转化为特征向量
  • 将此向量与分类器对比，得到最接近的分类结果
  • 将分类结果通过串口通信到UNO再由显示器输出（这一步对系统无实质性作用。因为想要练习软串口通信，就加上了：-））

硬件准备

  • Esp8266 Nodemcu
  • Arduino UNO
  • I2C LCD1602
  • 若干杜邦线

相关库

  • ESP8266
  • micromlgen （能生成可在微控制器上跑的归类代码）
  • LiquidCrystal_I2C (注意是RFRobot自己开发这个库,用于显示输出)

实现过程

步骤一：收集数据生成json格式文件

这一步只需要nodemcu就可以完成了。将以下代码放入Arduino的草图里。

#include <ESP8266WiFi.h>
#define print(string) Serial.print(string);
#define quote(string) print('"'); print(string); print('"');
String location = "";
void setup() {
  Serial.begin(115200);
  delay(3000);
  WiFi.disconnect();
}
void loop() {  
  // if location is set, scan networks
  if (location != ""&&(!Serial.available())) {
    int numNetworks = WiFi.scanNetworks();

    // print location
    print('{');
    quote("__location");
    print(": ");
    quote(location);
    print(", ");
  
    // print each network SSID and RSSI
    for (int i = 0; i < numNetworks; i++) {
      quote(WiFi.SSID(i));
      print(": ");
      print(WiFi.RSSI(i));
      print(i == numNetworks - 1 ? "}\n" : ", ");
    }
      
    delay(1000);
  }
  // else wait for user to enter the location
  else {
    String input;
    
    Serial.println("Enter 'scan {location}' to start the scanning");

    while (!Serial.available())
      delay(200);
      
    input = Serial.readStringUntil('\n');

    if (input.indexOf("scan ") == 0) {  //第一次 出现scan的时候是在字符串的第零位
      input.replace("scan ", "");
      location = input;
    }
    else {
      location = "";
    }
  }
}

在串口监测器里输入

scan 【你想要的扫描的房间】//进行扫描，如 scan room或者 scan 1

stop  //停止

这时串口监视器里会输出一些扫描到的数据（json格式）。在房间里走一走，等待一段时间后，可以进行下一个房间的扫描。最后，将数据整合放到一个json文件里。

步骤二：在spyder中生成分类代码（Python）

工具不限啦，只不过我用spyder比较多。
首先就是到cmd里安装库。

pip install micromlgen


然后将刚才得到的数据放进去

from micromlgen import port_wifi_indoor_positioning
from sklearn.tree import DecisionTreeClassifier
from micromlgen import port_wifi_indoor_positioning, port

if __name__ == '__main__':
    infile='替换为刚刚得到的json文件的路径'
    outfile_1='替换为路径\\Converter.h'
    outfile_2='替换为路径\\Classifier.h'
    with open(infile)as filedata:
       samples=filedata.read()

    X, y, classmap, converter_code = port_wifi_indoor_positioning(samples)
    clf = DecisionTreeClassifier()
    clf.fit(X, y)
    
    with open(outfile_1,'w')as out_1:
        out_1.write(converter_code)
    print(converter_code)
    with open(outfile_2,'w')as out_2:
        out_2.write(port(clf, classmap=classmap))
print(port(clf, classmap=classmap))

代码运行完后，你能在交互框中看到生成出来的特征转换器（ features converter）和分类器（ classifier）。同时也会保存到刚刚的路径里。

步骤四：UNO作为主控板接收数据

这一步主要是使用软串口接收nodemcu传过来的分类结果。
nodemcu与UNO连线：
D3->D3
D2->D2
VIN->VCC
GND->GND

这个例子中，我之前收集时输入偷懒了，使用的’1,2,3’来代替房间。这里用switch给了个映射，使输出到屏幕上的是实际的房间号。可以自己根据实际情况修改。

#include <SoftwareSerial.h>
#include <LiquidCrystal_I2C.h>
SoftwareSerial ArduinoUno(3,2);
LiquidCrystal_I2C lcd(0x20,16,2); 

void setup(){
  lcd.init();                 // LCD初始化设置
  lcd.backlight();           // 打开LCD背光
  Serial.begin(115200);
  ArduinoUno.begin(4800);
  lcd.print("You are in: ");
}

void loop(){
  while(ArduinoUno.available()>0){
    Serial.println("ready");
    int val=ArduinoUno.parseInt();
    if(ArduinoUno.read()=='\n'){
      Serial.println(val);
      lcd.setCursor(6,1); 
      switch (val){//根据实际情况修改
        case 1:
        lcd.print("304");break;
        case 2:
        lcd.print("207");break;
        case 3:
        lcd.print("311");break;
        default:
        lcd.print("outside");break;
      } 
    }
  }
  delay(100);
}

展望

其实对于大一点的房间，还是比较精准的。稍微想把空间划分的小一点，就不行了。
同时WiFi定位虽然对于硬件的要求不高（因为信号源都不需要自己准备），但是也正因如此会有很大的不确定性。例如附近wifi源过少或者wifi更变等不确定因素会使装备无法运转。
